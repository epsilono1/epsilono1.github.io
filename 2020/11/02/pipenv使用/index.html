    pipenv使用（windows）

    1. 安装
      pip install pipenv

    2. 快速创建并进入虚拟环境

      • 创建项目文件mypro，并cd进文件夹
      • 自定义虚拟环境路径（可选）
        • 项目文件夹下创建 .venv 文件夹
        • 创建虚拟环境 pipenv –three/two
      • 创建python3环境
        pipenv –three
      • 创建python2环境
        pipenv –two

    3. 修改包安装源

      • 编辑文件mypro/Pipfile
        • url = “https://pypi.org/simple" 替换成国内镜像
          以下任选一个：
          url = “https://pypi.tuna.tsinghua.edu.cn/simple/"
          url = “https://pypi.douban.com/simple/"
          url = “http://mirrors.aliyun.com/pypi/simple/"

    4. 在mypro目录下激活虚拟环境
      pipenv shell

    5. 安装库/包

       pipenv install requests==2.9.* #兼容的最新2.9.*，使用==时不要指定具体版本，否则无法更新
 pipenv install requests~=2.9.0 # >=2.9.0, <3.0; ==2.9.*
 pipenv install requests~=1.2 # >=1.2, <2.0; ==1.* 
 pipenv install mock-django~=0.6.10 # >=0.6.10, <0.7; ==0.6.*

    6. 查看已安装的包
      pipenv graph

    7. 删除包
      pipenv uninstall requests
      pipenv uninstall –all #删除全部

    8. 退出虚拟环境
      exit

    9. 扩展命令

      • 查看虚拟环境python所在位置
        pipenv –py
      • 查看虚拟环境位置
        pipenv –venv
      • 更改虚拟环境位置
        • 先在项目下创建 .venv文件夹（注意不是 .env）
        • 创建虚拟环境 pipenv –three
      • 查看所有需要更新的包
        pipenv update –outdated
      • 更新指定包
        • 更新前查看版本是否被 Pipfile 中限制住了
          比如最新4.9.3，Pipfile中是 ~= 4.8.0，那就更新不到4.9，需要把 ~= 4.8.0改大点
          pipenv update requests
      • 更新所有包
        pipenv update
      • requirements.txt导入
        • 如果项目自带requirements.txt，运行pipenv install命令时会自动导入
        • 其他位置导入
          pipenv install -r path/to/requirements.txt
      • requirements.txt导出
        pipenv lock -r
      • 查看python版本
        python –version
      • 更改python版本
        pipenv –python 3.6
        pipenv –python 3.7.4
        pipenv –python 2.7.14
      • 删除虚拟环境
        pipenv –rm
